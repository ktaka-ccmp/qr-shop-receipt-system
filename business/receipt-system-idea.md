# 店舗向け領収書自動発行システム

## 概要

店舗と利用者の双方がサービスに登録し、QRコードを介して電子領収書を発行・受領できるシステム。

## 登場人物

| 名称 | 説明 |
|------|------|
| 加盟店舗 | 領収書を発行する事業者（飲食店、小売店など） |
| 利用者 | 領収書を受け取る人（経費精算を行うビジネスパーソン等） |
| サービス事業者 | 本システムの運営者 |

## 基本フロー

```
1. 利用者が店舗に掲示されたQRコードを読み取る
2. サービスサイトにリダイレクト（未ログインならログイン発生）
3. 店舗情報が利用者のセッションに紐付く
4. 利用者のワンタイム参照トークン付きQRコードが自動表示される
5. 店員がQRコードを読み取り、金額・商品情報を入力して送信
6. サービスサイトで領収書が生成される
7. 利用者・店舗の双方から領収書を閲覧可能
```

## 技術要素

### ワンタイム参照トークン

- 短期間（数分程度）のみ有効なトークン
- サービスURLにトークンを付加してQRコード化
- トークンから利用者の基本情報（氏名・会社名等）を限定的に参照可能
- 店舗側はトークン経由でのみ利用者情報にアクセス（利用者IDは非公開）

### データ構造

領収書発行時に以下をセットで登録：

- 店舗ID
- 利用者ID
- 領収書ID
- 購入情報（金額、商品名、日時等）
- タイムスタンプ

### プライバシー設計

| 情報 | 店舗から | 利用者から |
|------|----------|------------|
| 相手のID | ✗ | ✗ |
| 相手の名前・会社名 | ○ | ○ |
| 領収書内容 | ○（自店発行分） | ○（自分宛分） |

## 利用者向け機能

### 領収書管理

- 受領した領収書の一覧表示・検索
- 個別領収書のPDFダウンロード
- 過去分の一括ダウンロード

### 会計連携

- 複数領収書を選択してCSVエクスポート
- 仕訳データ（勘定科目付き）として出力
- 主要会計ソフト向けフォーマット対応

### 勘定科目の自動判定

| 方法 | 内容 |
|------|------|
| デフォルト設定 | 飲食店→会議費、など業種別デフォルト |
| ルールベース | 「5,000円超は接待交際費」等の自然言語ルール |
| 履歴学習 | ユーザーの修正履歴からパターン学習 |
| AI補助 | 金額・時間帯・店舗種別から推論（補助的に） |

## 電子帳簿保存法対応

### 要件

電子取引データ保存として以下のいずれかを満たす必要あり：

- A. 認定タイムスタンプの付与
- B. 訂正削除履歴が残るシステム
- C. 訂正削除ができないシステム

### 実装方針

| フェーズ | 対応 |
|----------|------|
| プロトタイプ | 方法B：履歴テーブルで訂正削除履歴を保持 |
| 本格運用 | 方法A追加：認定タイムスタンプ＋電子署名 |

### タイムスタンプサービス候補

- セイコーソリューションズ（クロノトラスト）
- アマノタイムスタンプサービス
- サイバーリンクス（e-timing EVIDENCE）

## ビジネスモデル（B2B2C確定）

### 全体構造

```
本サービス → POSベンダー → 店舗 → 利用者
   (API)      (再販)      (発行)  (受領)
```

### 料金体系

#### 店舗側（POSベンダー経由）

| 項目 | 金額 | 備考 |
|------|------|------|
| 卸値（POSベンダー向け） | 10円/枚 | APIコールバックで課金 |
| 販売価格（店舗向け） | 15〜30円/枚 | POSベンダーが設定 |

**店舗メリット**: 5万円超の領収書1枚の印紙税200円と比較して圧倒的に安い

#### 利用者側（フリーミアム）

| プラン | 月額 | 機能 |
|--------|------|------|
| Free | 無料 | 領収書受領・閲覧（3ヶ月保存） |
| Light | 200円 | 全期間保存、PDF一括DL |
| Standard | 500円 | 仕訳CSV出力、勘定科目自動判定、会計ソフト連携 |

### 収益試算（1,000店舗、月100枚/店、有料化率20%）

- 店舗側: 10円 × 100枚 × 1,000店舗 = **100万円/月**
- 利用者側: 平均350円 × 2,000人 = **70万円/月**
- **合計: 170万円/月**

### 評価（7/10点）

**優れている点:**
- POSベンダー経由で一気にスケール可能
- APIコールバックで課金の正確性
- 店舗の印紙税削減メリットが明確

**懸念点:**
- POSベンダーの採用意欲（彼らにとっての旨味）
- 鶏と卵問題（店舗がないと利用者が来ない）
- 東芝テックが経費精算機能を追加した場合の競争

### 競合分析まとめ

**既存サービスの主なアプローチ：**
1. **「利用者がレシート撮影→OCR」モデル**（カテゴリC）
   - 普及している
   - OCRの精度問題あり
   - 店舗との連携なし

2. **「店舗が電子レシート発行」モデル**（カテゴリA）
   - スマートレシートが該当
   - 家計管理向け
   - 経費精算・仕訳出力に特化していない

**本アイデアとの差異：**

| 観点 | 既存サービス | 本アイデア |
|------|-------------|-----------|
| データ取得 | OCR（誤認識リスク） | 店舗から直接取得（正確） |
| 対象ユーザー | 家計管理 or 企業経理 | ビジネスパーソン個人の経費精算 |
| 仕訳機能 | なし or 別システム | 勘定科目自動判定＋CSV出力 |
| 電帳法対応 | 利用者側のみ | 店舗・利用者双方で原本保管 |
| 店舗メリット | 紙削減 | 紙削減＋印紙税不要＋顧客接点 |

## 差別化ポイント（競合調査後）

### 1. 店舗からの直接発行（OCR不要）
- 既存の経費精算システムはOCRによる読み取りが主流
- OCRは誤認識リスクがあり、確認・修正の手間が発生
- 本システムは店舗から直接データを受け取るため**100%正確**

### 2. 経費精算特化（スマートレシートとの差異）
- スマートレシートは家計管理向け
- 本システムは**ビジネスパーソンの経費精算**に特化
- 勘定科目の自動判定＋会計ソフト連携可能な仕訳CSV出力

### 3. 双方向の電子帳簿保存法対応
- 店舗側：発行した領収書の履歴管理
- 利用者側：受領した領収書の原本保管
- **税務対応を双方でサポート**

### 4. QRコードによるスムーズなUX
- 店舗QR読み取り → 自動ログイン → 利用者QR表示 → 発行
- 最小限の操作で完了

### 5. 店舗側のメリット明確化
- 紙の領収書・印紙税が不要（コスト削減）
- 発行履歴がデジタルで残る（監査対応）
- 顧客データ接点（マーケティング活用可能性）

- 本システムは独立サービスとして店舗に導入してもらう必要
- **POSレジ連携の優先度を上げる**ことが重要

- 競合ではなく**補完関係**として連携する道もある
- API連携で「店舗発行の電子領収書」を既存システムに取り込む
