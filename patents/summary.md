## 3. 高リスク特許の詳細

### 3.1 東芝テック 特許6598936

| 項目 | 内容 |
|------|------|
| **特許番号** | JP6598936B2 |
| **発明の名称** | 電子レシートシステム、端末装置およびプログラム |
| **出願日** | 2018年7月5日 |
| **有効期限** | **2034年9月5日**（予定） |
| **核心的構成** | フラグが付加された要素の**表示態様を変更**して表示 |

**回避策:**
- 電子領収書の特定項目をハイライト表示する機能を**実装しない**
- 色や背景の変更による項目強調機能を**実装しない**
- ポイント有効期限のハイライト表示を**実装しない**

### 3.2 東芝テック 特許6618585

| 項目 | 内容 |
|------|------|
| **特許番号** | JP6618585B2 |
| **発明の名称** | 電子レシートシステム（返品処理） |
| **出願日** | 2018年8月13日 |
| **有効期限** | **2033年9月6日**（予定） |
| **核心的構成** | 返品時の**訂正電子レシートデータ**の自動生成・送信 |

**回避策:**
- 返品処理と連動した電子レシートの自動訂正機能を**実装しない**
- 訂正電子レシートデータの自動生成・送信機能を**実装しない**

### 3.3 freee 特許5503795

| 項目 | 内容 |
|------|------|
| **特許番号** | JP5503795B2 |
| **発明の名称** | 自動仕訳機能（キーワード対応テーブル方式） |
| **出願日** | 2013年11月頃（推定） |
| **有効期限** | **2033年11月頃**（推定） |
| **核心的構成** | キーワードと勘定科目の**対応テーブル**を参照して自動的に仕訳 |

**回避策:**
- キーワード対応テーブル方式による勘定科目自動判定を**実装しない**
- **機械学習方式**は非侵害（freee vs マネーフォワード判決で確認済み）
- 勘定科目の**候補提示のみ**（利用者確定）は低リスク

## 4. 安全に実装可能な機能

### 4.1 低リスク機能

| 機能 | リスク | 根拠 |
|------|--------|------|
| 電子領収書データの生成・保存 | 低 | 一般的なDB操作 |
| QRコードによるトークン交換 | 低 | 基本特許は満了 |
| ユーザー端末への領収書データ配信 | 低 | 一般的なWeb技術 |
| 領収書データの統一フォーマット表示 | 低 | 経産省標準仕様あり |
| PDF/CSV形式でのダウンロード機能 | 低 | 一般的な技術 |
| 会計ソフト連携のためのデータエクスポート | 低 | API連携は一般的 |
| RFC3161準拠タイムスタンプ | 低 | 公開標準規格 |
| 訂正削除履歴の保存（履歴テーブル方式） | 低 | 一般的なDB設計 |

### 4.2 公知例による安全性の裏付け

| 技術 | 公知例 | 公開日 |
|------|--------|--------|
| 電子レシート基本機能 | スマートレシート | 2014年10月 |
| 標準フォーマット | 経産省標準仕様 | 2018年5月 |
| QRコード決済方式 | JPQR統一規格 | 2019年3月 |
| タイムスタンプ | RFC3161 | 2001年（RFC公開） |
| QRコード認証 | ロシアOFDシステム | 2017年7月 |

## 5. 推奨アクション

### 5.1 即時対応（設計フェーズ）

| 優先度 | アクション |
|--------|----------|
| **1** | ハイライト表示機能を設計から除外 |
| **2** | 勘定科目自動判定機能を設計から除外 |
| **3** | 返品連動機能を設計から除外（MVP以降も要検討） |

### 5.2 短期対応（1ヶ月以内）

| 優先度 | アクション |
|--------|----------|
| **4** | J-PlatPatでリクルート・スマレジ関連特許を詳細調査 |

| **6** | 認定タイムスタンプ事業者の選定 |

### 5.3 中期対応（3ヶ月以内）

| 優先度 | アクション |
|--------|----------|
| **7** | 会計ソフトAPI連携の設計（freee、MoneyForward） |
| **8** | 機械学習方式の勘定科目推定の技術検証（オプション） |
| **9** | 自社差別化機能の特許出願検討 |

## 6. フェーズ別実装方針

### Phase 1（MVP）

| 機能 | 実装 | 理由 |
|------|------|------|
| 電子領収書発行・保存 | **実装** | 低リスク |
| QRコードによる店舗-利用者連携 | **実装** | 低リスク（基本特許満了） |
| タイムスタンプ付与 | **実装** | 認定事業者API利用 |
| 訂正削除履歴管理 | **実装** | 一般的DB設計 |
| PDF/CSVエクスポート | **実装** | 低リスク |
| ハイライト表示 | **非実装** | 東芝テック特許 |
| 勘定科目自動判定 | **非実装** | freee特許 |

### Phase 2以降

| 機能 | 実装検討 | 条件 |
|------|---------|------|
| 会計ソフト連携 | 検討可 | API連携のみ（判定は会計ソフト側） |
| 勘定科目候補提示 | 検討可 | 弁理士確認後 |
| 機械学習による科目推定 | 検討可 | 十分なデータ蓄積後 |
| 返品連動機能 | 要慎重検討 | 東芝テック特許の権利範囲確認後 |
