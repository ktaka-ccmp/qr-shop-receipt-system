# 特許調査計画書

**作成日:** 2026年2月4日
**対象:** QR Shop Receipt System（電子領収書発行システム）

---

## 調査目的

1. 本ビジネスアイデアに関連する特許を網羅的に調査
2. 各特許の有効期限を確認し、特許切れの技術を特定
3. 請求項を解析し、ビジネスプランへの該当/非該当を判定
4. 回避すべき実装を明確化
5. 公知例（拒絶された出願、補正履歴等）を蓄積

---

### Phase 3: 特許有効期限確認

各特許について以下を確認：
- 出願日、登録日、存続期間満了日
- 年金納付状況（権利維持状況）
- 特許切れの技術を「安全圏」として分類

### Phase 4: 請求項詳細解析

高リスク特許について：
- 請求項の要件を分解・整理
- 本ビジネスプランの各機能との対比表作成
- 該当/非該当/要注意の判定

### Phase 5: 経過情報・公知例調査（特許データベース）

- 拒絶査定を受けた出願の調査
- 補正書・意見書の内容確認
- 審査過程で引用された先行技術文献の収集
- **引用特許の芋づる式調査**（引用文献→その引用文献→...とネットワーク構築）

### Phase 6: 公知例調査（特許データベース以外）

特許データベースに依存しない公知例の収集：

| 調査対象 | 具体例 | 公知例としての価値 |
|----------|--------|-------------------|
| 学術論文 | CiNii、Google Scholar、J-STAGE | 査読済みで信頼性高、日付明確 |
| 技術ブログ・記事 | Qiita、Zenn、企業技術ブログ | 実装例として有効、日付記録あり |
| 製品マニュアル・仕様書 | POSレジ、経費精算システムの公開ドキュメント | 実装の証拠 |
| プレスリリース | サービス発表時の報道 | 公開日が明確 |
| 標準化文書 | ISO、JIS等の規格書 | 公知例として強力 |
| オープンソースプロジェクト | GitHub等の類似実装 | コミット履歴で日付証明可能 |
| Internet Archive | Wayback Machineでの過去のWebサービス画面 | 日付付きアーカイブ |
| 展示会資料 | CEATEC、Japan IT Week等 | 公開日が明確 |
| 特許庁ガイドライン | 電子帳簿保存法関連のガイドライン | 公的文書 |

## サブエージェント作業分担

並列で以下の調査を実施：

### Agent 1: 東芝テック特許の深掘り調査
- 既知の特許（6598936, 6618585等）の経過情報確認
- 有効期限確認
- 他の関連特許の網羅的調査
- 拒絶された出願の調査

### Agent 2: freee特許の深掘り調査
- 自動仕訳関連特許の網羅的調査
- 経過情報確認
- 回避策の詳細検討

### Agent 3: POSレジベンダー（Airレジ、スマレジ、Square）
- リクルート、スマレジ、Square関連特許の調査
- 店舗QR表示方式の特許状況確認

### Agent 4: 電子帳簿保存法関連技術
- タイムスタンプ、訂正削除履歴関連特許
- 公知例の収集

### Agent 5: QRコード認証・決済関連
- QRコードを使った認証/決済の特許状況
- ワンタイムトークン関連特許

### Agent 6: 日立グループ特許調査
- 日立製作所、日立ソリューションズ、日立システムズの関連特許
- POS、決済、業務システム関連

### Agent 7: 引用特許の芋づる式調査
- 既知の重要特許が引用している先行技術文献を収集
- 引用文献がさらに引用している文献を追跡
- 引用ネットワークの構築

### Agent 8: 公知例調査（特許DB以外）
- 学術論文（CiNii、Google Scholar）
- プレスリリース、技術記事
- オープンソース実装
- Internet Archive（Wayback Machine）
