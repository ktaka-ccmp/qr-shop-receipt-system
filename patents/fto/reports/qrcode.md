## 1. QRコード基本特許の状況（デンソーウェーブ）

### 1.1 QRコードの歴史と特許状況

| 項目 | 内容 |
|------|------|
| **発明者** | 株式会社デンソーウェーブ（旧：日本電装） |
| **発明年** | 1994年 |
| **ISO規格化** | 2000年（ISO/IEC 18004） |
| **基本特許出願日** | 1994年頃 |
| **基本特許状態** | **満了（特許切れ）** |

### 1.2 デンソーウェーブの特許ポリシー

**重要：デンソーウェーブはQRコードの標準規格に関する特許権を行使しないと宣言している**

| 項目 | 内容 |
|------|------|
| **特許権不行使宣言** | JIS/ISO規格に準拠したQRコードの使用に対して特許権を行使しない |
| **宣言時期** | 1999年頃（規格化に際して） |
| **対象範囲** | QRコードの生成・読取に関する基本技術 |
| **制限** | 「QRコード」は登録商標であり、商標としての使用には許諾が必要な場合あり |

### 1.3 QRコード関連の主要特許（日本）

| 特許番号 | 名称（推定） | 出願日 | 状態 |
|---------|-------------|--------|------|
| 特許2938338 | 二次元コード（QRコード基本特許） | 1994年 | **満了** |
| 特許3716527 | マイクロQRコード関連 | 2001年頃 | **満了または満了間近** |
| 特許4048382 | 情報コード関連 | 2003年頃 | 2023年頃満了 |

**結論:** QRコードの基本特許は全て特許切れ（満了）であり、QRコードの生成・読取自体は自由に使用可能

### 1.4 QRコード商標について

| 項目 | 内容 |
|------|------|
| **「QRコード」商標** | 株式会社デンソーウェーブの登録商標 |
| **使用上の注意** | 商品名やサービス名として使用する場合は商標表示が推奨される |
| **表示例** | 「QRコードは株式会社デンソーウェーブの登録商標です」 |

## 2. QRコード認証関連特許

### 2.1 ワンタイムトークン/一時認証の技術分類

QRコードを使用した認証方式は以下のように分類される：

| 方式 | 概要 | 主な用途 | 特許リスク |
|------|------|----------|-----------|
| **静的QRコード** | 固定URLやIDを埋め込み | 製品識別、URL共有 | **低** |
| **動的QRコード** | 時間やセッションで変化 | 二要素認証、ログイン | **中〜高** |
| **ワンタイムQRコード** | 一度使用で無効化 | 決済、チケット | **中〜高** |
| **チャレンジレスポンス型** | サーバーとの双方向認証 | セキュア認証 | **中** |

### 2.2 主要なQRコード認証特許（推定）

| 権利者（推定） | 技術領域 | 特許リスク評価 |
|---------------|----------|---------------|
| Google | 二要素認証（Google Authenticator方式） | 低（TOTP/HOTPは公開仕様） |
| Microsoft | QRコードログイン | 中 |
| 各決済事業者 | 決済用ワンタイムQR | 高 |
| 東芝テック | 消費者コードによる電子レシート紐付け | **高（既知）** |

### 2.3 TOTP/HOTP（ワンタイムパスワード）の特許状況

| 仕様 | 説明 | 特許状況 |
|------|------|----------|
| **TOTP** | Time-based One-Time Password（RFC 6238） | **公開仕様・自由使用可** |
| **HOTP** | HMAC-based One-Time Password（RFC 4226） | **公開仕様・自由使用可** |

**本システムへの適用:**
- ワンタイムトークンの生成にTOTP/HOTP方式を採用することで、特許リスクを低減可能
- ただし、QRコードと組み合わせた具体的な実装方式には個別の特許が存在する可能性あり

### 2.4 本システムのQRコード認証フローと特許リスク

**本システムのフロー:**
```
[Step 1] 店舗がQRコード表示（店舗トークン）
    ↓
[Step 2] 利用者がQRコード読取
    ↓
[Step 3] 利用者がワンタイムトークンQR表示
    ↓
[Step 4] 店舗がQRコード読取 ← 【特許リスクあり】
    ↓
[Step 5] 電子領収書発行
```

| フロー部分 | 特許リスク | 根拠 |
|-----------|-----------|------|
| Step 1: 店舗QR表示 | 低 | 店舗表示→顧客読取は広く普及 |
| Step 2: 利用者読取 | 低 | 一般的な操作 |
| Step 3: 利用者QR表示 | **中〜高** | 東芝テック等の特許に類似 |
| Step 4: 店舗読取 | **高** | 消費者コード読取と同様の構成 |
| Step 5: 領収書発行 | 低 | 発行自体は一般的 |

| 項目 | 内容 |
|------|------|
| **JPQR** | 総務省主導の統一QRコード規格（2019年〜） |
| **目的** | 複数決済サービスを一つのQRコードで対応 |
| **特許状況** | 国が推進する規格であり、参加事業者間で特許のクロスライセンスが想定される |
| **本システムへの影響** | 決済機能を持たないため直接の関係なし |

### 3.4 主要決済事業者の特許保有状況（推定）

**結論:** 本システムは決済機能を持たないため、QRコード決済特許の直接的な侵害リスクは低い

### 3.5 決済特許との区別点

## 4. 消費者コード関連特許

### 4.1 「消費者コード」の定義

| 定義 | 説明 |
|------|------|
| **消費者コード** | 消費者（利用者）を識別するためのコード |
| **形式** | バーコード、QRコード、会員番号等 |
| **用途** | ポイント付与、電子レシート紐付け、会員識別 |

### 4.2 東芝テック特許における消費者コード

**特許6598936の構成要素:**

```
【請求項1】（抜粋）
...電子レシート情報を、当該取引に係る消費者を識別する消費者コードと
対応付けて記憶する記憶手段...
```

| 構成要素 | 東芝テック特許 | 本システム | 類似性 |
|---------|--------------|-----------|--------|
| 消費者識別 | 消費者コード（会員ID） | ワンタイムトークン | **類似** |
| コード読取 | 店舗POSが読取 | 店舗がQR読取 | **類似** |
| データ紐付け | 電子レシート | 電子領収書 | **類似** |

### 4.3 本システムとの比較

| 観点 | 東芝テック方式 | 本システム | 差異 |
|------|--------------|-----------|------|
| **コードの永続性** | 固定（会員ID） | **一時的（ワンタイム）** | **異なる** |
| **事前登録** | 必要（アプリ会員登録） | **不要** | **異なる** |
| **コード生成** | 事前に固定発行 | **都度生成** | **異なる** |
| **有効期間** | 無期限 | **数時間〜数日** | **異なる** |

### 4.4 ワンタイムトークンの技術的差別化

| 技術要素 | 特許回避の観点 | 評価 |
|---------|---------------|------|
| **トークンの一時性** | 固定IDとは異なる技術的特徴 | **差別化可能** |
| **有効期限管理** | 時間ベースのセキュリティ | **差別化可能** |
| **単回使用制限** | 一度の読取で無効化 | **差別化可能** |
| **事前登録不要** | アプリ会員登録なしで利用可能 | **差別化可能** |

**注意:** 上記の差別化要素が特許クレームの回避に十分かどうかは、弁理士による正式な鑑定が必要

| **利用者QRコード表示** | **中〜高** | 東芝テック特許との類似性 | フロー変更を検討 |

**現在のフロー（リスクあり）:**

```
   [低]          [低]          [中〜高]        [高]       [低]

```

## 9. 結論

### 9.1 特許リスク評価サマリー

| カテゴリ | リスクレベル | 対策 |
|---------|-------------|------|
| **QRコード基本技術** | **低（特許切れ）** | そのまま使用可 |
| **QRコード決済特許** | **低（決済機能なし）** | 非該当 |
| **消費者コード特許** | **高（東芝テック）** | フロー変更を推奨 |
| **ワンタイムトークン** | **低〜中** | TOTP/HOTP採用推奨 |

### 9.2 フロー変更の推奨

**現行案のリスク:**
「利用者がQRコード表示 → 店舗が読取」の工程は、東芝テック特許との類似性から**高リスク**と評価

**推奨される変更:**
- **Option A:** 利用者のQRコード表示を削除し、店舗QRのみの方式に変更
- **Option B:** QRコード以外の認証方式（メールアドレス入力、NFC等）を採用
- **Option C:** 弁理士鑑定でワンタイムトークン方式の安全性を確認後に実装

### 9.3 自由使用可能な技術

| 技術 | 状態 | 備考 |
|------|------|------|
| QRコード生成・読取 | **自由使用可** | JIS/ISO準拠 |
| TOTP/HOTPトークン | **自由使用可** | RFC公開仕様 |
| HTTPS/TLS通信 | **自由使用可** | 標準技術 |
| OAuth 2.0認証 | **自由使用可** | RFC公開仕様 |
| 一般的なWebAPI | **自由使用可** | REST/GraphQL |

### 9.4 最終推奨

1. **QRコード基本技術は安心して使用可能** - デンソーウェーブの特許は満了・不行使宣言済み

2. **「利用者QR表示→店舗読取」フローは要注意** - 東芝テック特許との類似性が高く、フロー変更を強く推奨

3. **決済関連特許は非該当** - 本システムは決済機能を持たないため、QRコード決済特許の直接的な侵害リスクは低い

4. **弁理士による正式鑑定が必須** - 本レポートは予備調査であり、最終判断は専門家に依頼すること
